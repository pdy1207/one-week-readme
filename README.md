# 프로젝트 개요

이 프로젝트는 마라톤 대회 참가 신청 시스템을 구축하는 웹 애플리케이션입니다.
참가자는 주어진 코스에서 선택하여 대회에 등록하며, 실시간 결제 정보 및 참가자 관리 기능을 포함합니다.

## 사용 버전

- **부트스트랩**: 5.3.8
- **부트스트랩 아이콘**: 1.11.1
- **jQuery**: 3.7.1
- **PHP**: 7.4.33
- **MySQL**: 5.6.51
- **WSL 사용**: Ubuntu-22.04
- **Nginx**: 1.29.7

## DB 구조

### 1:N 관계

- 참가자는 한 개의 대회에만 참여할 수 있으며, 검증은 휴대폰 번호로 진행됩니다.

## 대회 진행 흐름

### 메인 화면 (`index.php`)

- 코스별 정보를 제공하며, 참가자가 최대 인원에 도달하면 "마감" 표시가 나타납니다.
- 코스 설명에 따라 뱃지 색상이 달라집니다:
  - **가족 러닝**: `success` (그린)
  - **일반 코스**: `primary` (파랑)
  - **Half**: `warning` (노랑)
  - **Full**: `danger` (빨강)

### 대회 접수 절차

#### Step 1: 동의 화면 (`agree.php`)

- `agree.js`에서 동의 여부를 체크하여 값을 숨겨 전달합니다.
- 동의 여부를 `hidden` 필드를 통해 전달하며, 동의 값은 `0` (미동의) 또는 `1` (동의)로 구분됩니다.

#### Step 2: 코스 선택 (`cors.php`)

- `agree.js`에서 전달된 동의 값 검증 후, 코스 목록을 API로 불러와 바인딩합니다.
- 코스 카드를 선택하면 결제 예정 금액이 실시간으로 업데이트됩니다.

#### Step 3: 정보 입력 (`info.php`)

- 동의 값과 선택한 코스 값을 `queryString`으로 전달합니다.
- `sessionStorage`에 저장된 데이터가 있을 경우 복원하여 새로고침 시에도 데이터가 유지되도록 합니다.
- 코스 정보는 `readonly`로 설정되며, 배경색은 선택된 코스에 맞게 변경됩니다.
- `queryString` 값의 임의 조작 시, 세션이 삭제되고 대회 현황 페이지로 리다이렉트됩니다.
- 2008년 4월 7일 이후 출생자는 참가가 제한됩니다.

#### Step 4: 최종 제출 (`pay.php`)

- info.php에서 전달받은 데이터를 기반으로 최종 신청자 정보를 구성하고 화면에 표출합니다.
- JavaScript의 `Math.random()`을 활용하여 **2~3초 랜덤 지연**을 발생시킵니다.
- 해당 랜덤 로직에서 **10% 확률로 실패 케이스**를 생성합니다.
- 정상 완료 시, 신청 데이터를 **POST 방식으로 JSON 형태로 전송**합니다.

- **참가자 번호 생성 규칙**
  - `SPR` + `26` + `A-Z`, `0-9` 랜덤 조합
  - 예시: `SPR26-91AZGD`

- **정원 체크 및 동시성 처리**
  - 코스 ID 기준으로 현재 참가자 수를 카운트합니다.
  - 코스 테이블에서 최대 인원을 조회합니다.
  - 기존에는 `FOR UPDATE`를 통해 동시 업데이트를 방지했습니다.
  - 현재 인원이 최대 인원 이상일 경우 **마감 처리**됩니다.

#### Step 5: 접수 내역 조회 (`list.php`)

- 등록 시와 동일한 검증 로직을 수행합니다.
- 이름과 휴대폰 번호를 기준으로 신청 데이터를 조회합니다.
- 조회된 결과를 화면에 표 형태로 마크업하여 출력합니다.
- 조회 결과가 없을 경우 예외 처리를 통해 안내 메시지를 표시합니다.

## 관리자(Admin) 시스템 구성

#### Step 1: 로그인 (login.php)

- 관리자에게 사전에 발급된 **아이디 / 비밀번호**로 로그인
- 비밀번호는 `password_hash()`로 해시 처리 (bcrypt 알고리즘)
- 로그인 시 `password_verify()`를 사용하여 검증

#### 처리 흐름

1. 아이디로 사용자 조회
2. 저장된 해시 비밀번호와 입력값 비교
3. 일치 시 로그인 성공 (세션 생성)


#### Step 2: 대시보드 (admin_dashboard.php)

- 로그인한 관리자 이름을 상단에 표시
- 전체 현황을 한눈에 확인 가능

#### 주요 지표

- 총 접수 인원
- 결제 완료 인원
- 코스별 참여 비율
- 사이즈별 현황
- 대회 접수 및 발주 현황 (토탈 통계)

#### Step 3: 참가자 상세 (details.php)

- 참가자 정보 리스트 출력

#### 표시 항목

- 참가번호
- 이름
- 코스
- 연락처
- 결제 상태
- 사이즈
- 주소
- 접수 일시

#### 기능

- 코스 필터 기능
- 이름 검색 기능
- 페이지네이션
  - 기본: 10개 표시
  - 더보기 버튼 클릭 시 5개씩 추가 로딩

#### Step 4: 메시지 발송 (send_message.php)

- 코스 목록 및 전체 사용자 데이터를 불러와 대상 설정 가능
- 선택한 대상에게 문자 또는 이메일 발송

#### 문자 발송

- 알리고, 솔라피, 카카오 알림톡 등을 고려했으나
- 템플릿 검수 기간 문제로 미적용

#### 이메일 발송

- Gmail SMTP로 테스트 진행했으나
- 설정 및 발송 문제로 인해 적용 미완료

#### 참고 사항

- 발송 기능은 외부 API 또는 SMTP 연동이 필요
- 대량 발송 시 성능 및 비용 고려 필요
- 테스트 발송 기능 별도 구현 권장
